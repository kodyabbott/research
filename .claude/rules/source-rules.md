# Source Verification Rules

- Every factual claim needs a source: URL, file path with line number, or paper citation.
- Quotes must be verbatim. If paraphrasing, don't use quotation marks.
- If a quote is attributed to a person, verify it against the primary source (video, transcript, article with direct quotes). Circulating paraphrases drift.
- If you can't verify a claim, mark it with **TODO** in notes.md and add a caveat in README.md.
- After drafting, review each claim. Find a supporting quote. If you can't, remove the claim and mark with `[REMOVED -- no supporting source found]`.
- Star counts, version numbers, and statistics go stale. Use approximate values with dates: "~500 stars as of April 2026."
- Don't attribute someone's paraphrase to the original source. If Simon paraphrased ThoughtWorks, cite it as Simon's paraphrase, not ThoughtWorks' claim.
